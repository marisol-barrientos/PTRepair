A resolution strategy may consist of one or more process change operations. If no feasible strategy exists, explicitly state this and explain why.

When generating the resolution strategy:

- Use only the provided process change operations.
- Prefer minimal and localized modifications.
- Avoid unnecessary complexity or structural inconsistencies.
- If full compliance cannot be achieved, generate the best partial-compliance strategy.
- Ensure that multiple operations remain valid when applied sequentially.
- Provide a brief `change_description`.
- Assess the overall `change_risk` (`very_low`, `low`, `medium`, `high`, or `very_high`) and briefly justify it.

A set of resolution context requirements is also provided. These requirements are currently satisfied and should remain satisfied whenever possible.

- Avoid introducing new violations of the resolution context requirements.
- Prefer strategies that preserve already compliant behavior.
- Minimize and justify any unavoidable trade-offs.

Return only valid JSON conforming to the provided resolution strategy schema. Do not include markdown, code fences, or additional text.
